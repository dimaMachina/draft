Mintlify turns MDX files and a single JSON config into a polished, production-ready documentation site. Follow the steps below to go from zero to a live local preview — no prior Mintlify experience required.

<Note>
  You need **Node.js 18 or later** installed before you begin. Run `node -v` in your terminal to confirm your version.
</Note>

## Set Up Your Docs Site

<Steps>
  <Step title="Install the Mintlify CLI">
    Install the CLI globally so you can scaffold and preview docs from any directory on your machine.

    <CodeGroup>
      ```bash npm
      npm install -g mintlify
      ```

      ```bash yarn
      yarn global add mintlify
      ```

      ```bash pnpm
      pnpm add -g mintlify
      ```
    </CodeGroup>

    Confirm the installation succeeded by checking the version:

    ```bash Terminal
    mintlify --version
    ```
  </Step>

  <Step title="Scaffold a new project">
    Run the `init` command in the directory where you want your docs to live. Mintlify creates the folder structure and a starter `docs.json` config for you.

    ```bash Terminal
    mintlify init my-docs
    cd my-docs
    ```

    <Tip>
      You can also run `mintlify init` inside an existing repository to add docs alongside your source code.
    </Tip>

    The generated `docs.json` controls your site's name, navigation, colours, and more. Open it and update the basics:

    ```json docs.json
    {
      "name": "My Product Docs",
      "logo": {
        "light": "/logo/light.svg",
        "dark": "/logo/dark.svg"
      },
      "favicon": "/favicon.svg",
      "colors": {
        "primary": "#0D9373",
        "light": "#07C983",
        "dark": "#0D9373"
      },
      "navigation": [
        {
          "group": "Get Started",
          "pages": ["introduction", "quickstart"]
        }
      ]
    }
    ```
  </Step>

  <Step title="Add your first content page">
    Create an MDX file for any page listed in `docs.json`. Every page starts with a frontmatter block that sets the title and description shown in search results and the browser tab.

    ```jsx introduction.mdx
    ---
    title: "Welcome to My Docs"
    sidebarTitle: "Introduction"
    description: "Everything you need to get up and running with My Product."
    ---

    Welcome to the official documentation. Use the sidebar to explore guides,
    API references, and component examples.

    <Note>
      This site is built with [Mintlify](https://mintlify.com) — the docs
      platform for modern developer tools.
    </Note>
    ```

    <Warning>
      The filename (without `.mdx`) must match the string you used in the `pages` array inside `docs.json`. A mismatch causes a 404 in the sidebar.
    </Warning>
  </Step>

  <Step title="Preview locally">
    Start the local development server from inside your docs directory. Mintlify watches for file changes and hot-reloads the browser automatically.

    ```bash Terminal
    mintlify dev
    ```

    Open [http://localhost:3000](http://localhost:3000) in your browser to see your docs site. Edit any `.mdx` file or `docs.json` and the preview updates instantly.

    <Tip>
      Pass `--port 4000` to run the server on a different port if 3000 is already in use: `mintlify dev --port 4000`.
    </Tip>
  </Step>

  <Step title="Deploy to production">
    Push your docs to a GitHub (or GitLab) repository, then connect the repo to your Mintlify dashboard. Mintlify automatically rebuilds and deploys on every push to your default branch.

    ```bash Terminal
    git init
    git add .
    git commit -m "docs: initial Mintlify setup"
    git remote add origin https://github.com/your-org/your-repo.git
    git push -u origin main
    ```

    After connecting the repository in the [Mintlify dashboard](https://dashboard.mintlify.com), your docs are live at `https://your-subdomain.mintlify.app` within seconds.
  </Step>
</Steps>

## Configuration Quick Reference

Use these tabs to explore the most common `docs.json` configuration sections.

<Tabs>
  <Tab title="Navigation">
    Group your pages into labelled sections. Each group appears as a collapsible section in the sidebar.

    ```json docs.json
    "navigation": [
      {
        "group": "Get Started",
        "pages": ["introduction", "quickstart"]
      },
      {
        "group": "Components",
        "pages": [
          "components/callouts",
          "components/code",
          "components/cards"
        ]
      }
    ]
    ```
  </Tab>
  <Tab title="Branding">
    Set your logo, favicon, and brand colours. Mintlify applies these across the entire site, including syntax highlighting accents.

    ```json docs.json
    "logo": {
      "light": "/logo/light.svg",
      "dark": "/logo/dark.svg"
    },
    "favicon": "/favicon.svg",
    "colors": {
      "primary": "#6366F1",
      "light": "#818CF8",
      "dark": "#4F46E5"
    }
    ```
  </Tab>
  <Tab title="Top Navigation">
    Add links to the top navbar — useful for directing readers to your main site, GitHub, or changelog.

    ```json docs.json
    "topbarLinks": [
      {
        "name": "Support",
        "url": "https://example.com/support"
      }
    ],
    "topbarCtaButton": {
      "name": "Dashboard",
      "url": "https://dashboard.example.com"
    }
    ```
  </Tab>
</Tabs>
