Tabs let you present parallel content — different frameworks, operating systems, or configuration styles — within the same vertical space. Instead of forcing readers to scroll past content that doesn't apply to them, you surface only what's relevant through a clean tab switcher. Use `<Tabs>` any time you have two or more complete content variants that address the same task.

<Tip>
  **Tabs vs. CodeGroup:** Use `<Tabs>` when each variant contains a full section of content (prose, callouts, lists, or code). Use `<CodeGroup>` when you're switching between code-only variants of the same snippet. Tabs are the heavier-weight option — reach for `<CodeGroup>` first if you only need to swap code blocks.
</Tip>

## Basic Tabs — Same Concept, Different Frameworks

The example below shows how to define a reactive counter component in three popular front-end frameworks. Each tab presents the same concept using framework-native syntax.

<Tabs>
  <Tab title="React">
    In React, you manage local state with the `useState` hook. The component re-renders automatically whenever `count` changes.

    ```jsx Counter.jsx
    import { useState } from "react";

    export default function Counter() {
      const [count, setCount] = useState(0);
      return (
        <button onClick={() => setCount(count + 1)}>
          Clicked {count} times
        </button>
      );
    }
    ```
  </Tab>
  <Tab title="Vue">
    In Vue 3, use the Composition API's `ref` function to declare reactive state. The template updates reactively whenever the ref value changes.

    ```vue Counter.vue
    <script setup>
    import { ref } from "vue";
    const count = ref(0);
    </script>

    <template>
      <button @click="count++">Clicked {{ count }} times</button>
    </template>
    ```
  </Tab>
  <Tab title="Svelte">
    Svelte compiles reactivity away at build time. Declare a plain variable and Svelte tracks mutations automatically — no import needed.

    ```svelte Counter.svelte
    <script>
      let count = 0;
    </script>

    <button on:click={() => count++}>
      Clicked {count} times
    </button>
    ```
  </Tab>
</Tabs>

```jsx Basic Framework Tabs

  </Tab>
  <Tab title="Vue">
    In Vue 3, use the Composition API's ref function.
  </Tab>
  <Tab title="Svelte">
    Svelte compiles reactivity away at build time.
  </Tab>
</Tabs>
```

## Tabs With API Code Blocks

Use tabs to show the same API request written in different languages. This pattern lets developers copy the version that matches their stack without scrolling past irrelevant snippets.

<Tabs>
  <Tab title="JavaScript">
    ```js Send a Message
    const response = await fetch("https://api.example.com/v1/messages", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        Authorization: `Bearer ${process.env.API_KEY}`,
      },
      body: JSON.stringify({
        recipient: "user_01HXYZ",
        content: "Hello from the API!",
      }),
    });

    const data = await response.json();
    console.log(data.message_id);
    ```
  </Tab>
  <Tab title="Python">
    ```python send_message.py
    import os
    import requests

    response = requests.post(
        "https://api.example.com/v1/messages",
        headers={
            "Authorization": f"Bearer {os.environ['API_KEY']}",
        },
        json={
            "recipient": "user_01HXYZ",
            "content": "Hello from the API!",
        },
    )

    data = response.json()
    print(data["message_id"])
    ```
  </Tab>
  <Tab title="cURL">
    ```bash Send a Message
    curl -X POST https://api.example.com/v1/messages \
      -H "Content-Type: application/json" \
      -H "Authorization: Bearer $API_KEY" \
      -d '{
        "recipient": "user_01HXYZ",
        "content": "Hello from the API!"
      }'
    ```
  </Tab>
</Tabs>

```jsx API Code Block Tabs
<Tabs>
  <Tab title="JavaScript">
    JavaScript fetch code here
  </Tab>
  <Tab title="Python">
    Python requests code here
  </Tab>
  <Tab title="cURL">
    curl command here
  </Tab>
</Tabs>
```

## Tabs for Platform-Specific Instructions

When your installation or setup process differs across operating systems, use tabs to keep all platforms in one place. Readers pick their OS once and see exactly the steps they need.

<Tabs>
  <Tab title="macOS">
    Install the Mintlify CLI using npm:

    ```bash macOS Install
    npm install -g mintlify
    ```

    Then verify the installation:

    ```bash Verify
    mintlify --version
    ```

    If you see `command not found`, ensure `/usr/local/bin` is in your `PATH` by adding the following line to your `~/.zshrc`:

    ```bash ~/.zshrc
    export PATH="/usr/local/bin:$PATH"
    ```
  </Tab>
  <Tab title="Windows">
    Open **PowerShell as Administrator** and install the Mintlify CLI via npm:

    ```powershell Windows Install
    npm install -g mintlify
    ```

    Verify the installation:

    ```powershell Verify
    mintlify --version
    ```

    If npm global binaries aren't recognized, add the npm global directory to your system `PATH` through **System Properties → Environment Variables**.
  </Tab>
  <Tab title="Linux">
    Install the Mintlify CLI globally with npm:

    ```bash Linux Install
    npm install -g mintlify
    ```

    Verify the installation:

    ```bash Verify
    mintlify --version
    ```

    On some distributions the global npm prefix requires `sudo`. To avoid this, configure a user-local prefix instead:

    ```bash Configure npm prefix
    mkdir -p ~/.npm-global
    npm config set prefix '~/.npm-global'
    echo 'export PATH="$HOME/.npm-global/bin:$PATH"' >> ~/.bashrc
    source ~/.bashrc
    ```
  </Tab>
</Tabs>

```jsx Platform-Specific Tabs
<Tabs>
  <Tab title="macOS">

  </Tab>
  <Tab title="Windows">
    npm install -g mintlify (from PowerShell as Administrator)
  </Tab>
  <Tab title="Linux">
    npm install -g mintlify
  </Tab>
</Tabs>
```

## Tab Synchronization

When multiple `<Tabs>` or `<CodeGroup>` blocks on the same page share **identical tab labels**, Mintlify synchronizes them automatically. Selecting "Python" in one block switches every other block with a "Python" tab to match.

<Note>
  Tab sync is scoped to the current page and persists across page navigation via `localStorage`. If a reader selects "Python" on one page, they'll see "Python" pre-selected when they arrive on any other page that has a matching tab label.
</Note>

For example, if your page has both of the following blocks and a reader picks **Python** in the first, the second automatically switches to **Python** too:

<Tabs>
  <Tab title="JavaScript">
    ```js Install SDK
    npm install @example/sdk
    ```
  </Tab>
  <Tab title="Python">
    ```bash Install SDK
    pip install example-sdk
    ```
  </Tab>
</Tabs>

<Tabs>
  <Tab title="JavaScript">
    ```js Initialize Client
    import { ExampleClient } from "@example/sdk";
    const client = new ExampleClient({ apiKey: process.env.API_KEY });
    ```
  </Tab>
  <Tab title="Python">
    ```python Initialize Client
    from example_sdk import ExampleClient
    client = ExampleClient(api_key=os.environ["API_KEY"])
    ```
  </Tab>
</Tabs>

```jsx Synchronized Tabs
<Tabs>
  <Tab title="JavaScript">
    npm install @example/sdk
  </Tab>
  <Tab title="Python">
    pip install example-sdk
  </Tab>
</Tabs>

<Tabs>
  <Tab title="JavaScript">
    import ExampleClient and initialize it with your API key.
  </Tab>
  <Tab title="Python">
    from example_sdk import ExampleClient and initialize it.
  </Tab>
</Tabs>
```
