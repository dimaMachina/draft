Code blocks are one of the most important elements in technical documentation. Mintlify renders fenced code blocks with full syntax highlighting out of the box, and extends standard Markdown with a set of meta options — written after the language tag — that let you add titles, line numbers, diffs, focused lines, and more. This page walks through every option with a live example and the exact MDX source that produces it.

---

## Basic Code Block

Start with a standard fenced code block. Add a language tag immediately after the opening triple backtick to enable syntax highlighting.

```javascript
function greet(name) {
  return `Hello, ${name}!`;
}

console.log(greet("world"));
```

## Code Block with a Title

Add a title (filename, terminal name, or label) directly after the language tag, separated by a space. The title appears as a tab at the top of the block.

```python src/main.py
def calculate_tax(amount: float, rate: float) -> float:
    """Calculate tax for a given amount and rate."""
    return round(amount * rate, 2)

total = calculate_tax(199.99, 0.08)
print(f"Tax owed: ${total}")
```

## Line Numbers

Add the `lines` meta option to display line numbers in the gutter. This is especially helpful when your prose refers to a specific line ("on line 4, notice that…").

```javascript index.js lines
import express from "express";

const app = express();
const PORT = process.env.PORT || 3000;

app.get("/", (req, res) => {
  res.json({ status: "ok", message: "Server is running" });
});

app.listen(PORT, () => {
  console.log(`Listening on port ${PORT}`);
});
```

## Highlighted Lines

Use the `highlight` option to draw attention to specific lines. Pass a single line number, a range, or a comma-separated list. Highlighted lines appear with a distinct background color.

```typescript utils/auth.ts lines highlight={5,9-11}
import jwt from "jsonwebtoken";

const SECRET = process.env.JWT_SECRET!;

export function signToken(payload: object): string {
  return jwt.sign(payload, SECRET, { expiresIn: "7d" });
}

export function verifyToken(token: string): object | null {
  try {
    return jwt.verify(token, SECRET) as object;
  } catch {
    return null;
  }
}
```

## Focused Lines

Use the `focus` option to dim every line except the ones you specify. This is useful when you want the reader to concentrate on a particular section of a longer snippet.

```python pipeline.py focus={6-9}
import pandas as pd
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression

pipeline = Pipeline([
    ("scaler", StandardScaler()),
    ("classifier", LogisticRegression(max_iter=200)),
])

pipeline.fit(X_train, y_train)
score = pipeline.score(X_test, y_test)
print(f"Accuracy: {score:.2%}")
```

## Diff Markers

Use `// [!code ++]` and `// [!code --]` inline comments to mark added and removed lines. Mintlify renders them as a green/red diff, just like a Git diff view.

```javascript config.js
export default {
  host: "localhost", // [!code --]
  host: "0.0.0.0", // [!code ++]
  port: 8080,
  debug: true, // [!code --]
  debug: process.env.NODE_ENV !== "production", // [!code ++]
  timeout: 5000,
};
```

## CodeGroup — Multi-Language Tabs

Wrap multiple code blocks in a `<CodeGroup>` to render them as tabbed panels. Label each tab with the title meta on each block. This is perfect for showing the same API call in multiple languages.

<CodeGroup>
```javascript Node.js
const response = await fetch("https://api.example.com/v1/users", {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    Authorization: `Bearer ${process.env.API_KEY}`,
  },
  body: JSON.stringify({ name: "Alice", email: "alice@example.com" }),
});

const user = await response.json();
console.log(user.id);
```

```python Python
import httpx

response = httpx.post(
    "https://api.example.com/v1/users",
    headers={"Authorization": f"Bearer {API_KEY}"},
    json={"name": "Alice", "email": "alice@example.com"},
)

user = response.json()
print(user["id"])
```

```bash cURL
curl -X POST https://api.example.com/v1/users \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $API_KEY" \
  -d '{"name": "Alice", "email": "alice@example.com"}'
```
</CodeGroup>

## CodeGroup with Dropdown

Add the `dropdown` prop to `<CodeGroup>` to render the language switcher as a compact dropdown menu instead of a full tab bar. This saves horizontal space when you have many languages.

<CodeGroup dropdown>

const client = new Client({ apiKey: process.env.API_KEY });
const result = await client.items.list({ limit: 10 });

## Meta Options Reference

The table below lists every supported meta option you can add after the language tag on a fenced code block.

| Option | Type | Description | Example |
|---|---|---|---|
| `title` | string | Display name shown in the code block header tab | `bash Terminal` |
| `lines` | flag | Show line numbers in the left gutter | `javascript lines` |
| `highlight` | range | Emphasize specific lines with a background tint | `highlight={3,7-10}` |
| `focus` | range | Dim all lines except the specified range | `focus={5-9}` |
| `expandable` | flag | Collapse the block behind a "Show more" toggle | `json expandable` |
| `wrap` | flag | Wrap long lines instead of enabling horizontal scroll | `bash wrap` |
| `twoslash` | flag | Enable TypeScript TwoSlash type annotations | `typescript twoslash` |

<Note>
  You can combine multiple meta options on the same line. For example, `typescript utils/token.ts lines highlight={4-6}` adds a filename, line numbers, and line highlighting all at once.
</Note>
