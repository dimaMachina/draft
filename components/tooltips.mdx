Tooltips let you embed a short definition directly inside your prose so readers can hover over an unfamiliar term and get an instant explanation — without navigating away from the page or hunting through a separate glossary. In Mintlify, a tooltip is a single `<Tooltip>` component that wraps any word or phrase and accepts your definition through the `tip` prop.

## Basic Tooltip Syntax

Wrap the term you want to annotate in a `<Tooltip>` tag and pass your definition to the `tip` prop. Keep the definition short — one phrase or sentence is ideal.

```jsx title="Basic Tooltip usage"
Use <Tooltip tip="A markup language for writing Mintlify docs">MDX</Tooltip> to write your pages.
```

**Live example:**

Use <Tooltip tip="A markup language for writing Mintlify docs">MDX</Tooltip> to write your pages.

Hover over the highlighted word above to see the tooltip appear. The wrapped text receives a subtle underline or highlight so readers know it is interactive.

## Tooltips in Paragraph Context

Tooltips shine when you are writing a technically dense paragraph and want to keep it readable for both experts and newcomers. Rather than adding a footnote or breaking the flow with a parenthetical, you annotate each term in place.

```jsx title="Multiple tooltips in a paragraph"
Every Mintlify page is an <Tooltip tip="MDX is Markdown with embedded JSX components">MDX</Tooltip> file
that begins with a <Tooltip tip="A YAML block at the top of a file that sets page metadata like title and description">frontmatter</Tooltip> block.
Inside the file you compose your content using built-in
<Tooltip tip="A reusable JSX element you drop into MDX, such as Card, Callout, or Accordion">components</Tooltip>,
each referenced by its <Tooltip tip="The URL-friendly identifier for a page, derived from its file path">slug</Tooltip>
in the navigation configuration.
```

**Live example:**

Every Mintlify page is an <Tooltip tip="MDX is Markdown with embedded JSX components">MDX</Tooltip> file that begins with a <Tooltip tip="A YAML block at the top of a file that sets page metadata like title and description">frontmatter</Tooltip> block. Inside the file you compose your content using built-in <Tooltip tip="A reusable JSX element you drop into MDX, such as Card, Callout, or Accordion">components</Tooltip>, each referenced by its <Tooltip tip="The URL-friendly identifier for a page, derived from its file path">slug</Tooltip> in the navigation configuration.

<Note>
  Tooltips are accessibility-friendly on desktop — they work with both mouse hover and keyboard focus. However, on mobile devices there is no hover state, so tooltip content may be inaccessible to mobile readers. For truly critical definitions, consider also linking to a dedicated glossary page.
</Note>

## Tooltips vs Linking to a Separate Page

Both tooltips and links give readers access to more information, but they serve different needs. Use the guidance below to decide which is right for a given term.

| Situation | Recommendation |
|---|---|
| Brief one-line definition (under 80 characters) | `<Tooltip>` — keeps the reader on the page |
| Complex concept that needs several paragraphs | Link to a dedicated page or expandable `<Accordion>` |
| Term that appears many times across many pages | Create a glossary page and link to it |
| Acronym or jargon with a universally short expansion | `<Tooltip>` — the expansion is the whole definition |
| A concept the reader must understand to continue safely | Inline callout (`<Warning>` or `<Note>`) — don't hide critical info in a hover |

```jsx title="Tooltip vs page link — choosing the right pattern"
{/* ✅ Good use of tooltip — short, self-contained definition */}
Configure your <Tooltip tip="A JSON file at the root of your project that controls navigation and theme">docs.json</Tooltip> file.

{/* ✅ Good use of link — the concept deserves a full explanation */}
Learn how [API authentication](/api-reference/authentication) works before you go live.
```

## Tooltip Best Practices

<AccordionGroup>
  <Accordion title="Keep tooltip text under 80 characters">
    Tooltip bubbles are narrow. If your definition runs longer than roughly 80 characters it will wrap awkwardly or overflow on smaller screens. Edit ruthlessly: strip articles, cut examples, and reduce to the single most useful fact. If you cannot fit the definition in 80 characters, the concept probably deserves its own page.

    ```jsx title="Short vs long tooltip text"
    {/* ✅ Concise — fits comfortably in the bubble */}
    <Tooltip tip="A reusable JSX element embedded in MDX">component</Tooltip>

    {/* ❌ Too long — consider a link to a dedicated page instead */}
    <Tooltip tip="A reusable JSX element that you embed directly inside your MDX files to add rich interactive UI such as cards, tabs, callouts, and accordions">component</Tooltip>
    ```
  </Accordion>

  <Accordion title="Use tooltips for jargon, acronyms, and technical terms">
    Tooltips work best on terms that are opaque to a subset of your audience but obvious to others — acronyms like `MDX`, `YAML`, or `SDK`, domain-specific jargon like "frontmatter" or "slug", and product-specific concepts unique to your platform. Avoid adding tooltips to plain English words that any reader will understand — it creates noise and trains readers to ignore the highlights.

    ```jsx title="Good tooltip candidates"
    Your <Tooltip tip="Application Programming Interface — a set of endpoints your code calls">API</Tooltip> key is scoped per workspace.

    Set the <Tooltip tip="How long a browser caches a response, in seconds">TTL</Tooltip> to 3600 for hourly expiry.
    ```
  </Accordion>

  <Accordion title="Don't use tooltips for critical information">
    A reader who misses a tooltip hover misses the information entirely. Never put safety warnings, destructive action notices, required configuration steps, or anything labelled "important" inside a tooltip. Use a `<Warning>` or `<Note>` callout for content the reader must see.

    ```jsx title="Critical info belongs in a callout, not a tooltip"
    {/* ❌ Critical warning hidden in a tooltip — easy to miss */}
    Click <Tooltip tip="This action is irreversible and deletes all data">Delete</Tooltip>.

    {/* ✅ Critical warning in a visible callout */}
    <Warning>
      Deleting a workspace is **irreversible**. All data will be permanently removed.
    </Warning>
    ```
  </Accordion>

  <Accordion title="Prefer a link to a full page for complex concepts">
    When a concept has nuance, exceptions, or requires examples to understand properly, a tooltip will under-explain it and may actively mislead readers who take the short definition too literally. In those cases, link to a dedicated page or an `<Accordion>` that has the space to do the concept justice.

    ```jsx title="Link instead of tooltip for complex concepts"
    {/* ❌ Over-simplified tooltip for a complex topic */}
    <Tooltip tip="Controls how your site looks">docs.json</Tooltip>

    {/* ✅ Link to the full reference */}
    See the [API reference overview](/api-reference/overview) for every available endpoint and configuration option.
    ```
  </Accordion>
</AccordionGroup>
