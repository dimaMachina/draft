Mintlify uses [Font Awesome 6](https://fontawesome.com/icons) as its icon library. Wherever a Mintlify component accepts an `icon` prop, pass the kebab-case name of any Font Awesome icon — for example `"rocket"`, `"bolt"`, or `"puzzle-piece"` — and Mintlify renders the corresponding glyph automatically. Icons are entirely optional, but they give readers visual anchors that make long navigation lists and reference pages significantly easier to scan.

<Note>
  Icons are optional on every component that supports them. Skip the `icon` prop if your content is text-heavy or if adding icons would distract rather than clarify. When in doubt, use icons consistently across a whole section rather than mixing icon and icon-free items in the same list.
</Note>

## Icons in Card Components

Pass the `icon` prop on any `<Card>` to display an icon above the card title. Use a `<CardGroup>` to lay multiple icon cards out in a responsive grid.

<CardGroup cols={3}>
  <Card title="Getting Started" icon="rocket">
    Ship your first doc in minutes with Mintlify's quickstart guide.
  </Card>
  <Card title="Performance" icon="bolt">
    Learn how to keep your documentation site fast for every reader.
  </Card>
  <Card title="API Reference" icon="code">
    Auto-generate interactive API reference pages from an OpenAPI spec.
  </Card>
  <Card title="Guides" icon="book">
    Step-by-step walkthroughs for common documentation patterns.
  </Card>
  <Card title="Configuration" icon="gear">
    Customise navigation, themes, and site settings in `docs.json`.
  </Card>
  <Card title="Security" icon="shield">
    Protect docs with password authentication or SSO.
  </Card>
</CardGroup>

## Icons in Accordion Components

Add `icon` to any `<Accordion>` to display an icon next to the accordion's title in both its collapsed and expanded states.

<AccordionGroup>
  <Accordion title="Authentication" icon="key">
    Configure API key, OAuth 2.0, or JWT authentication for your endpoints. Each method uses a different `securitySchemes` entry in your OpenAPI spec.
  </Accordion>
  <Accordion title="CLI Reference" icon="terminal">
    The Mintlify CLI lets you preview your docs locally, check for broken links, and deploy changes without leaving your terminal.
  </Accordion>
  <Accordion title="Integrations" icon="puzzle-piece">
    Connect Mintlify to Slack, GitHub, Segment, and dozens of other tools through the integrations dashboard.
  </Accordion>
</AccordionGroup>

## Icons in Code Block Headers

Add an `icon` meta option directly to a fenced code block to render a small icon in the code block's title bar. Combine it with `title` to give the block a labelled, visually distinct header.

````mdx title="Code block with icon meta"
```bash icon="terminal" title="Install the Mintlify CLI"
npm install -g mintlify
```
````

```bash icon="terminal" title="Install the Mintlify CLI"
npm install -g mintlify
```

<Tip>
  Browse the full Font Awesome 6 Free library at [fontawesome.com/icons](https://fontawesome.com/icons). Filter by **Free** and **Solid** to see every icon available in Mintlify. </Tip>
